# DTblocklyGPT: Cobotta Digital Twin

## 🛠️ Environment & Tech Stack
- **OS:** Ubuntu 24.04 (Optimized for ARM64 via UTM on Apple Silicon)
- **Robotics Middleware:** ROS 2 Jazzy
- **Simulation:** Gazebo Harmonic
- **Backend:** Python / Django / Flask
- **Frontend:** React / Vite / Blockly

---

## 🚦 Startup Guide

Because this is a distributed system, you need to launch the components in parallel using **6 separate terminal windows**. 
*Note: Ensure you run the commands in the specified order to prevent synchronization crashes.*

### 1. The Backend (Django)
Manages the REST APIs, SQLite database, and OpenAI integrations.
```bash
cd ~/DTblocklyGPT
poetry run python manage.py runserver
```

### 2. The Frontend (React/Vite)
The user interface featuring the Blockly visual workspace.
```bash
cd ~/DTblocklyGPT/frontend
npm start
```
*(Access the app at `http://localhost:3000`)*

### 3. The Simulator (Gazebo)
Calculates kinematics and physics. 

**Option A: Headless Mode (Recommended for logic testing & saving CPU)**
```bash
cd ~/DTblocklyGPT/ros2_ws/Cobotta
gz sim -s worldCobotta.sdf
```

**Option B: Full 3D GUI Mode (Run directly from the Ubuntu native terminal)**
```bash
cd ~/DTblocklyGPT/ros2_ws/Cobotta
gz sim worldCobotta.sdf
```

### 4. The ROS-Gazebo Bridge
Translates ROS 2 topics to internal Gazebo transport topics.
```bash
cd ~/DTblocklyGPT/ros2_ws
source install/setup.bash
ros2 run ros_gz_bridge parameter_bridge --ros-args -p config_file:=./Cobotta/map.yaml
```

### 5. The Robotic Brain (ROS 2 Node)
The core controller that calculates trajectories and moves the Cobotta joints.
```bash
cd ~/DTblocklyGPT/ros2_ws
source install/setup.bash
ros2 run cobotta_rest_api cobotta_node
```

### 6. The HTTP-to-ROS Bridge (Flask Node)
Exposes an HTTP API on port `:5000`. Django uses this to send simulation commands generated by Blockly/ChatGPT directly to the ROS network.
**⚠️ Crucial for the "Simulate" button to work in the GUI.**
```bash
cd ~/DTblocklyGPT/ros2_ws
source install/setup.bash
ros2 run cobotta_rest_api flask_node
```
